# AI Shop OS 部署形态与使用方式

## 1. 当前系统以什么形式存在

当前 AI Shop OS 是一套 Web 控制系统，由三部分组成：

- 前端控制台：老板、客服和运营人员在浏览器里使用。
- 后端 API：负责权限、平台接入、导入任务、AI 工作流和数据读写。
- Supabase 数据库：保存商家、店铺、订单、商品、客户、消息、售后、记忆和学习记录。

它不是一个单独插件，也不是一个聊天窗口。更准确地说，它是“AI电商员工的后台工作台”。

## 2. 老板怎么用

老板主要使用：

- 老板首页：看当天销售、退款、异常、建议和审批。
- AI员工：查看 AI客服、AI售后、AI运营等员工状态。
- 平台集成：接入淘宝、抖音、闲鱼等店铺数据。
- 数据导入：上传闲鱼或平台导出的 CSV/Excel 文件。
- 审批：确认退款、赔偿、投流预算和高风险动作。

老板不需要每天看几十张报表，系统目标是让 AI 先分析，再让老板审批。

## 3. AI客服怎么用

AI客服用于处理客户消息。

当前版本支持：

- 平台消息收件箱。
- AI 回复草稿。
- 自动回复状态。
- 人工接管。
- 商家修改记录。
- 学习样本采集。

当前不做：

- 擅自退款。
- 擅自赔偿。
- 擅自承诺物流结果。
- 擅自处理投诉。

## 4. AI售后怎么用

AI售后用于处理退款、退货、物流异常、投诉和赔偿。

当前版本支持：

- 售后 case 列表。
- 风险等级。
- AI 判断。
- 审批建议。
- 处理记录。

高风险动作必须老板或客服主管确认。

## 5. AI运营怎么用

AI运营用于增长、获客和投流，但必须分阶段上线。

第一阶段建议只做：

- 商品数据分析。
- 高意向客户识别。
- 私域引导话术草稿。
- 优惠券建议。
- 爆款商品线索。
- 投流计划草稿。

暂不允许：

- 自动花钱投广告。
- 自动改预算。
- 自动上架/下架商品。
- 自动改商品价格。

涉及预算和平台规则的动作必须审批。

## 6. 接入商店的正确顺序

推荐顺序：

1. 配置 Supabase 数据库。
2. 部署后端 API。
3. 部署前端控制台。
4. 接入淘宝官方授权。
5. 接入抖音官方授权。
6. 使用文件导入闲鱼数据。
7. 导入订单、商品、客户、物流和售后。
8. 让客服在 AI客服工作台处理真实消息。
9. 收集 300 到 1000 条客服修正样本。
10. 再开启更高自动化能力。

## 7. 上线前必须完成的配置

必须配置：

- Supabase 项目。
- PostgreSQL 数据库连接 `DATABASE_URL`。
- Supabase Auth。
- Redis。
- 后端 API 域名。
- 前端网站域名。
- 平台 OAuth 回调地址。
- 令牌加密密钥。

不能把下面内容暴露到前端：

- Supabase service role key。
- 淘宝 app secret。
- 抖音 app secret。
- 令牌加密密钥。
- 数据库密码。

## 8. 私域获客投流的安全边界

AI可以做分析、建议、草稿和提醒。

AI不能直接做高风险动作。

允许：

- 识别高意向客户。
- 生成私域引导话术。
- 建议优惠券策略。
- 输出投流计划。
- 输出素材方向。
- 提醒老板审批。

必须审批：

- 广告预算。
- 优惠金额。
- 私域群发。
- 投诉处理。
- 退款赔偿。
- 客户数据导出。

## 9. 当前距离正式上线还差什么

当前已经有：

- 前端页面。
- AI客服工作台。
- AI售后工作台。
- 订单、商品、客户表格。
- 平台数据导入入口。
- 淘宝、抖音、闲鱼接入结构。

仍需完成：

- Supabase migration 真正执行。
- `DATABASE_URL` 配置真实密码。
- Redis/RQ Worker 部署。
- 后端部署到公网。
- 前端部署到公网。
- 淘宝、抖音真实开放平台凭证。
- 首家商家真实数据联调。

## 10. 推荐部署方式

前端：

- Vercel，适合 Next.js。

后端：

- Railway、Render、Fly.io 或云服务器 Docker。

数据库：

- Supabase PostgreSQL。

队列：

- Redis + RQ Worker。

文件：

- Supabase Storage。

## 11. 一句话总结

AI Shop OS 是一个浏览器里的 AI 电商员工控制台：老板审批，AI工作，客服修正，系统学习，所有高风险动作都可追踪。
